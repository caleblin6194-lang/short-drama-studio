0 阅读指南

本 PRD 是从竞品（纳米 AI 短剧工具）的 7 步线性流水线重构而来。原版本的逻辑骨架是对的（资产库先行 + 非线性回改），但 UI 是为懂行的人设计的，新手转化率低。本版本把 7 步压成 4 阶段，在不牺牲专业能力的前提下大幅降低新手认知门槛，并加入一项关键差异化能力：一稿多投。

阅读顺序建议：

  1. 第 1 章 — 看清楚我们要做什么和为什么
  2. 第 2 章 — 看整体架构
  3. 第 3 章 — 看数据模型（工程的脊梁）
  4. 第 4-8 章 — 按阶段看交互和接口细节
  5. 第 9-12 章 — 跨阶段机制和工程考虑
  6. 第 13 章 — MVP 怎么砍刀
  7. 第 14 章 — 怎么验收

1 产品定位

1.1 一句话定位

让没拍过短剧的人，点四下、等七分钟，做出一集可以直接发抖音/红果/ReelShort 的 AI 短剧；让会拍的人，五分钟做一集，十分钟做五个语种版本。

1.2 核心价值主张

门槛低到底

不需要懂分镜、不需要懂模型、不需要懂画面比例。会写故事就行，不会写故事 AI 帮你起头。

资产复用强

角色/场景/道具一次建库，跨镜头跨集数跨地域复用。

非线性回改

任何一步都能回去改，下游自动同步。

一稿多投

同一剧本一键生成国内、北美、东南亚、日韩、中东 5 个市场的本地化版本。所有竞品都没有。

1.3 与纳米 AI 7 步流水线的差异

维度纳米原版本产品
步骤数7 个独立 Tab4 个连续阶段
首屏负担全局设定要先选 6+ 个技术参数4 行标签点 4 下，参数自动反推
剧本起点空白编辑器AI 预生成 3 个开场白可选
资产管理独立 Tab "资产库"伪装成"剧组筹备会"审片流程
分镜编辑一屏摊开 10+ 控件大预览 + 一句话改镜 + 高级折叠
视频/口型Step 5 / Step 6 独立 Tab折叠为镜头流水线的内部阶段
出海支持无一稿多投到 5 个市场

1.4 目标用户

P0 主要用户 — 新手创作者

短视频创作者

看过短剧但没拍过，每天可投入 1-3 小时

对 AI 工具有基本使用经验（用过 ChatGPT / 即梦 / 可灵）

不懂镜头语言、不知道九宫格机位、不在乎模型策略

核心诉求：做出能发出去、有人看的内容

P1 次要用户 — 出海 MCN / 短剧工作室

出海团队

已有国内短剧拍摄经验

想低成本试水海外市场，但不懂当地文化

核心诉求：同一剧本快速做多语种版本，看哪个市场跑得动

非目标用户

专业影视团队

院线级动画团队、广告 4A、专业影视公司

他们会用 Runway / Midjourney 配 After Effects，本工具的天花板对他们太低

2.3 非线性回改原则

任何阶段都能跳回上游修改，工程层面必须实现失效传播。
  • 改 TagSet → InferredConfig 重算 → 用户可选是否重生成所有资产
  • 改场景资产 → 引用该场景的所有 Shot 自动标记为 stale，提示用户重生成
  • 改某 Shot 的画面描述 → 该 Shot 的 image / video / audio 全部置为 pending
  • 改某 Shot 的图 → 该 Shot 的 video / audio 置为 pending（下游依赖）
  • 改某 Shot 的视频 → 该 Shot 的 audio 置为 pending

4.3 开场白预生成

用户每次切换标签组合，后台异步调用 LLM 生成 3 个完全不同的开场白，缓存 5 分钟。超时 8 秒降级为模板兜底文案。

4.4 参考剧库

为什么重要：出海创作者最大的盲点是用国内套路拍海外剧。让他在选完地域的瞬间就能看到"哦原来北美在看狼人吸血鬼，不是赘婿"，胜过任何教程。

4.5 剧本编辑器

  • 富文本编辑器，支持撤销/重做/复制/清空
  • 自动保存（每 3 秒 debounced）
  • 字数计数（默认上限 10000）
  • "AI 续写"按钮：基于当前剧本调用 LLM 生成接下来 200-300 字
  • 导入剧本功能：.txt / .md / .docx

5 阶段二：剧组筹备会

5.1 剧本解析

用户点"下一步"时，后端调用 LLM 解析剧本，识别出所有场景、角色、道具。5-15 秒完成。

5.2 资产卡组件

类型卡片宽高比视觉权重
场景4:3大卡片，4 列网格
角色3:4大卡片，4 列网格
道具横向 chip极小，inline 排列

资产卡状态机

pending ──[进入阶段二]──> generating
generating ──[成功]──> ready
generating ──[失败]──> failed
ready ──[通过]──> approved (布尔标记，不是状态)
ready ──[换一张/微调]──> generating
failed ──[重试]──> generating
关键决策：approved 是独立布尔标记，不是状态机的一个状态。用户即使没有审核完，也能跳到阶段三。审核是可选精修，不是必经卡口。

5.3 资产生成 Pipeline

  • 同一项目内最多 4 个并发生成任务
  • 每个资产生成完立即通过 WebSocket 推送给前端
  • 前端按事件逐一替换占位卡为成品卡（进度感）

5.4 一致性机制

工程上最难的一块。"一处修改全剧同步"听起来简单，实现需要角色一致性 token（IP-Adapter face embedding 或 LoRA 权重）。

MVP 降级：如果 P0 来不及做，可以接受"角色脸会跑偏"的事实，UI 上不主动承诺一致性。

6 阶段三：开机拍摄

6.1 镜头数据初始化

进入阶段三时，后端基于剧本 + 资产库调用 LLM 生成镜头列表。所有 Shot 的 pipeline 状态为 pending。

6.2 自动开拍调度器

流水线并行：S01 在做 video 的同时，S02 在做 image，S03 在排队。这不是简单串行，而是 3-stage pipeline parallelism。

6.4 一句话改镜

这是阶段三的主路径，80% 的用户修改请求通过这一个输入框完成。
用户写自然语言如"眼神再凶一点"或"换个机位拍背影"，后端调 LLM 重写 description，然后重跑整个 pipeline。

9 跨阶段机制

9.1 失效传播

修改的字段受影响的下游处理
TagSetInferredConfig自动重算
InferredConfig.castEthnicity所有 Character 资产提示用户重生成
InferredConfig.artStyle所有 Scene 资产提示用户重生成
Script.rawTextShot 列表 + 资产识别弹框："剧本变了，要重新解析吗？"
Asset.imageUrl引用该资产的 Shot pipeline.image自动置 stale
Shot.description该 Shot 的 image/video/audio自动置 pending
Shot.dialogue该 Shot 的 audio自动置 pending
MasterCut configrenderedUrldebounced 重渲染
关键原则：
  1. 不静默重生成——需要调用模型的失效都应提示用户确认（生成是付费动作）
  2. Stale 可视化——虚线边框 + 黄色"已过期"角标
  3. 批量重生成——用户可"一键更新所有过期内容"

9.2 自动保存与版本快照

  • 任何输入框修改 debounced 1 秒自动保存
  • 每次重大动作生成 ProjectVersion 快照
  • 快照保存最近 20 个，允许回滚

9.3 LLM 调用统一封装

所有 LLM 调用走内部 LLMGateway 服务：厂商抽象、Prompt 版本化、Token 计费、失败重试 + 降级、Response schema validation。

13 风险与对策

一致性风险

问题：角色脸在不同镜头间会跑偏
对策：P0 接受不完美 → 提供"重生成"按钮让用户挑 → P1 上 LoRA → 长期训练自有模型

成本风险

问题：LLM + 图像 + 视频 + 配音叠加成本可能超付费意愿
对策：严格并发控制 + 失败不重复扣费 + 智能模型路由 + 持续优化 prompt

模型能力天花板

问题：AI 视频模型做不到电影级质量
对策：定位短剧（高密度切镜 + 强情绪对白） + 强化字幕/BGM/音效包装 + 快速接入新模型

本地化质量

问题：LLM 翻译水土不服
对策：cultural_notes 让用户审核 + "保守/激进"两档 + P1 人工 reviewer + 数据反馈优化 prompt

合规风险

问题：不同地域内容合规要求不同
对策：各地域"禁止/敏感"关键词表 + negative prompt 注入 + 导出前内容审查 + 高风险人工审核

15 术语表

术语含义
TagSet用户在阶段一选择的 4 个创作标签 + 受众
InferredConfig由 TagSet 自动反推出来的技术参数
资产库 (AssetLibrary)项目内所有场景、角色、道具的集合
镜头 (Shot)最小视频片段，5-8 秒，引用 1 个场景 + 若干角色道具
流水线 (Pipeline)单个 Shot 的三阶段生成：image → video → audio
一句话改镜用自然语言修改单个 Shot 的主交互方式
自动开拍阶段三核心动作，按顺序为所有 Shot 跑流水线
成片 (MasterCut)所有 Shot 拼接 + 片头尾 + 字幕 + BGM 的完整一集
一稿多投同一项目以另一地域市场的本地化版本重新生成
RegionVariant一稿多投产物，共享剧本骨架，独立本地化资产和成片
失效传播上游修改导致下游产物失效的自动标记机制
一致性 token角色跨图像保持外观的技术手段（embedding 或 LoRA）
积分工具的计费单位，所有生成动作消耗积分
