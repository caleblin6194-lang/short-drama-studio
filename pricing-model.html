1.3 竞品计费模式总结

  • 订阅 + 积分/Credit 混合制是主流模式（Runway、Pika、可灵、Hailuo 均采用）

9 我们的计费方案设计

9.1 计费模式选择

决策：采用「订阅 + 积分」混合制。
  • 订阅制提供稳定的基础额度 + 高级功能解锁（去水印、高画质、商用授权）
  • 积分制让用户按需消费，避免"用不完浪费"的心理负担
  • 与竞品（Runway、可灵、Hailuo）的模式一致，用户教育成本低

9.2 积分体系设计

9.5 定价逻辑

定价公式：用户支付价 = 实际模型成本 × 3-5 倍加价率
  • cost_first 单集实际成本 ≈ ¥14 → 用户消耗约 ¥14 × 3 = ¥42 积分（~1400 积分）
  • 创作者套餐 ¥49/月给 5000 积分 → 够做 ~3.5 集 → 单集用户成本 ¥14
  • 这意味着创作者套餐的用户感知是"¥14/集"，远低于任何真人短剧的单集制作成本（通常 ¥3000+）
  • 加价率覆盖：模型成本 + 平台运维 + 算法迭代 + 合理利润

11 风险与成本优化

11.1 成本控制策略

短期（P0）
  • 并发控制：每用户最多 4 个并发生成任务
  • 失败不扣费：模型调用失败的积分原路返还
  • 缓存复用：开场白/参考剧库缓存 5 分钟
  • prompt 优化：减少 token 浪费，提高一次成功率
  • 降级链：贵模型失败自动 fallback 到便宜模型
中期（P1-P2）
  • MuseTalk 自部署：替代 Hedra 口型同步，成本降 80%
  • SDXL 自部署：图片生成成本趋近于零
  • Batch API：非实时任务走 OpenAI Batch（50% off）
  • Gemini Context Caching：重复 prompt 场景降 90%
  • 动态路由：监控各厂商价格变化，实时切换最优

重要发现：按当前成本结构，积分定价覆盖不了模型成本。这是 AI 应用的普遍困境。

解法：
  1. 提高积分消耗：视频生成从 30 积分提至 50 积分（cost_first 单集 ~870 积分 → ¥8.7）
  2. 套餐定价上调：创作者套餐 ¥79/月、工作室 ¥299/月更合理
  3. 用免费额度做漏斗，靠付费加购盈利：积分加购的加价率更高（~5x）
  4. 中期自部署降成本：视频模型自部署后成本可降 50-70%
  5. 广告/分发分成：P2 阶段接入平台分发，从内容收益中抽成
